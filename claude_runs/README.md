# claude_runs/ — created by Claude

Everything in this folder was generated by Claude to **demonstrate the simulation
pipeline end-to-end without touching your original files**. Nothing here overwrites
`../result/` or `../plot/` (your originals, dated earlier, are untouched).

## Contents
- `config_smoke.yaml` — a copy of `../config.yaml` with `save_dir` redirected here.
- `result/` — checkpoints from the demo run (`sim_model_hidden8_fold0.pt`, `autosave_...`).
- `plot/` — figures from `plot_smoke.py` (`neural_recon.png`, `states.png`).
- `plot_smoke.py` — a copy of `../plot.py` that reads/writes inside `claude_runs/`.
- `train_log.txt` — stdout from the training run.

## How it was run
Used the existing `sleap-new` conda env (it already has torch 2.8.0 + numpy + sklearn +
matplotlib), so no new environment had to be built:

```bash
# train (CPU, ~3 s/epoch on this machine)
conda run -n sleap-new python array_hidden8.py \
    --config claude_runs/config_smoke.yaml --fold 0 --epochs 300

# plot
conda run -n sleap-new python claude_runs/plot_smoke.py
```

## Caveat
This is a **short demonstration run** (300 epochs vs. the paper's 2000) on CPU, just to
prove the pipeline works and produce example figures. Reconstruction/state-recovery
quality will be lower than a full run — do the full 2000-epoch run (ideally on
HiPerGator, see [../docs/05_hipergator_guide.md](../docs/05_hipergator_guide.md)) for
real results.
